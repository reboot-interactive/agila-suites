Xenon Suite

Technical Documentation & Deployment Guide

v1.2 — March 2026

Overview

Xenon Suite is a centralized ERP (Enterprise Resource Planning) system built on Laravel 12. It serves as the single source of truth for product catalog data, inventory, and order management across multiple sales channels, including OpenCart storefronts, Shopee, Lazada, and TikTok Shop.

Centralized Catalog

Manage products, categories, manufacturers, and options from one place with OpenCart-compatible schema.

Multi-Store OpenCart Sync

Two-way sync with multiple OpenCart stores. Push product data out, pull orders and stock in.

Shopee Integration

OAuth authentication, order sync, return handling, stock push, and product listing management.

Lazada Integration

OAuth authentication, order sync, reverse orders, pack/ship fulfillment, and product upload.

TikTok Shop

OAuth authentication, API explorer, and foundational integration for TikTok Shop marketplace.

Reports & Analytics

Order reports, product performance, payout tracking, category analytics, fee breakdowns, and inventory reports.

Deployment Guide

Step-by-Step Deployment

Clone the Repository 
git clone git@github.com:your-org/xenon-suite.git /var/www/xenon-suite
cd /var/www/xenon-suite
Install PHP Dependencies 
composer install --optimize-autoloader --no-dev
Configure Environment 
cp .env.example .env

Edit .env and configure at minimum:

APP_NAME="Xenon Suite"
APP_ENV=production
APP_DEBUG=false
APP_URL=https://erp.yourdomain.com

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=xenon_suite
DB_USERNAME=your_db_user
DB_PASSWORD=your_db_password

QUEUE_CONNECTION=database
SESSION_DRIVER=database
CACHE_STORE=database
Generate Application Key 
php artisan key:generate
Run Database Migrations 
php artisan migrate --force
Build Frontend Assets 
npm install
npm run build
Set File Permissions 
chmod -R 775 storage bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache
Configure Web Server

Point your web server document root to the public directory.

Apache (.htaccess included)

Ensure mod_rewrite is enabled. Laravel ships with a public/.htaccess file.

Nginx

server {
    listen 80;
    server_name erp.yourdomain.com;
    root /var/www/xenon-suite/public;

    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}
Set Up Cron for Scheduled Tasks 
* * * * * cd /var/www/xenon-suite && php artisan schedule:run >> /dev/null 2>&1
Configure Queue Worker

For processing queued jobs (order sync, stock push, etc.), run a queue worker as a supervised process:

php artisan queue:listen --tries=1 --timeout=0

For production, use Supervisor or systemd:

[program:xenon-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/xenon-suite/artisan queue:work --sleep=3 --tries=3 --max-time=3600
autostart=true
autorestart=true
stopasgroup=true
killasgroup=true
user=www-data
numprocs=1
redirect_stderr=true
stdout_logfile=/var/www/xenon-suite/storage/logs/worker.log
Quick Setup Script The project includes a composer setup script that automates steps 2-6 (install, .env copy, key generation, migration, and frontend build).
Local Development For local development, run composer dev to start the dev server, queue listener, Pail log viewer, and Vite dev server concurrently.

Middleware & RBAC

EnsurePermission Middleware

The EnsurePermission middleware (registered as perm) enforces role-based access control. It is applied to route groups via perm:permission_key syntax.

Route::middleware(['perm:manage_products'])->group(function () {
    // Only users with manage_products permission can access
});

Multiple permissions can be specified with a pipe (|) separator. Any one match grants access:

Route::middleware(['perm:manage_shopee|manage_lazada'])->group(function () { ... });

Permission Hierarchy

Permissions follow a parent-child hierarchy. Granting a parent permission automatically includes all children. Having any child permission satisfies the parent check for sidebar visibility.

Marketplace Integrations

Shopee

Authentication (OAuth 2.0)

Shopee uses OAuth 2.0 with partner-level credentials. Configure partner_id and partner_key in the Shopee settings page, then use the authorization flow to obtain shop-level access tokens.

  • Authorize: GET /shopee/authorize — Redirects to Shopee for shop authorization
  • Token Get: POST /shopee/token/get — Exchange auth code for access token
  • Token Refresh: POST /shopee/token/refresh — Refresh expired access token

Features

  • Order Sync: Automatic and manual order sync with configurable date range. Creates/updates orders in unified order table.
  • Return Handling: Sync return/refund records from Shopee with separate status mapping.
  • Stock Push: Push ERP product quantities to Shopee for all linked products via shopee:push-stock.
  • Product Listing: Link catalog products to Shopee items. Sync product IDs, quantities, and prices.
  • Profiles: Create listing profiles with category templates, attributes, and logistics channel assignments.
  • Category Browser: Fetch and browse the Shopee category tree with attribute details.
  • Logistics: View and manage Shopee logistics/shipping channels.
  • Order Status Mapping: Map Shopee statuses (UPPERCASE) to ERP order statuses. Separate mappings for order and return contexts.
  • API Explorer: Test arbitrary Shopee API calls directly from the UI.
  • API Logging: Toggle request/response logging for debugging.
  • Sandbox Mode: Toggle between production and sandbox environments.

Lazada

Authentication (OAuth 2.0)

Lazada uses region-based OAuth with an app key and app secret. The callback URL (/lazada/callback) is publicly accessible to receive the authorization code.

  • Authorize: GET /lazada/authorize — Redirects to Lazada for authorization
  • Callback: GET /lazada/callback — Receives auth code (public route)
  • Token Create: POST /lazada/token/create — Exchange auth code for tokens
  • Token Refresh: POST /lazada/token/refresh — Refresh expired access token

Features

  • Order Sync: Automatic order sync with configurable date range. Creates/updates orders in unified order table.
  • Reverse Orders: Sync return/refund/cancellation records from Lazada.
  • Pack & Ship: Pack orders, create packages, set "Ready to Ship", generate AWB labels, packing lists, and pick lists.
  • Order Cancellation: Cancel orders with reason selection.
  • Logistics Trace: Track shipment progress for individual orders.
  • Stock Push: Push ERP product quantities to Lazada for linked products via lazada:push-stock.
  • Product Upload: Create new products on Lazada with full attribute, variant, and image support.
  • Profiles: Create listing profiles with category templates, manufacturer filtering, and dual markup (percentage + fixed).
  • Category/Brand Tools: Browse Lazada categories, fetch attributes, query brands.
  • Order Status Mapping: Map Lazada statuses (lowercase) to ERP order statuses. Separate mappings for order and reverse contexts.
  • API Explorer: Test arbitrary Lazada API calls from the UI.
  • Sandbox Mode: Toggle between production and sandbox environments.

OpenCart Multi-Store Sync

Overview

Xenon Suite supports bi-directional synchronization with multiple OpenCart stores. Each store has its own connection settings, API key, and sync timestamps.

OpenCart API Module

The ERP includes a custom OpenCart 2.3 module that exposes a REST API for Xenon Suite to communicate with. This module runs on the OpenCart catalog (frontend) side to avoid admin login requirements.

Deployment

Copy the 5 files from opencart_extensions/opencart-api-module/ to your OpenCart installation, maintaining the directory structure:

Copy files to your OpenCart root 
# From ERP project root:
cp -r opencart_extensions/opencart-api-module/* /path/to/opencart/
Enable the module in OpenCart admin

Go to Extensions > Modules > ERP Sync API. Enable the module, set an API key, and optionally configure an IP whitelist and debug logging.

Configure the ERP

In Xenon Suite, go to Marketplace > OpenCart, add a store, and enter the store URL and the API key you configured.

Authentication

All endpoints require the X-ERP-API-Key header. The API key is configured in the OpenCart admin module settings. Optional IP whitelist support is available.

     https://yourstore.com/index.php?route=api/erp/ping

Troubleshooting

Migration Failures

IssueCauseSolution
Table already existsPartial migration runRun php artisan migrate:status to check state. If stuck, manually mark completed migrations in the migrations table.
SQLSTATE[42000] syntax errorMySQL version too oldEnsure MySQL 8.0+ or MariaDB 10.6+. Some catalog tables use fulltext indexes that require InnoDB support.
Foreign key constraint failureTables created out of orderRun php artisan migrate:fresh on new installations. On existing databases, check that referenced tables exist.
Prefix-related errorsDB_TABLE_PREFIX not set correctlyVerify DB_TABLE_PREFIX in .env matches your OpenCart table prefix (e.g., oc_).

API Authentication Errors

IssueCauseSolution
401 Unauthorized on API callsMissing or invalid Sanctum tokenEnsure the Authorization: Bearer {token} header is included. Tokens expire when the user logs out.
419 CSRF token mismatchMissing CSRF token on web routesInclude X-CSRF-TOKEN header for POST/PUT/DELETE requests made via the web middleware.
403 ForbiddenUser lacks required permissionCheck the user's group permissions. The EnsurePermission middleware checks against the permissions table via the user's group.

Marketplace Sync Issues

IssueCauseSolution
Shopee/Lazada "Missing credentials"OAuth tokens not configured or expiredGo to the marketplace settings page and complete the OAuth flow. Then run the token refresh command.
Shopee token refresh failsRefresh token expired (30+ days)Re-authorize via /shopee/authorize to get fresh tokens.
Lazada token refresh failsRefresh token expiredRe-authorize via /lazada/authorize to get fresh tokens.
Orders not syncingSync date range misconfiguredCheck the "sync last N days" setting on each marketplace's settings page. Increase the range or run a manual fetch.
Stock not pushingProducts not linkedEnsure products are linked to marketplace items via product_links tables. Check for unmatched items.
OpenCart connection failedAPI module not deployed or disabledVerify the ERP Sync API module is installed and enabled in OpenCart. Check the API key matches. Use the Test Connection button.
OpenCart API key mismatchKey in ERP doesn't match OpenCart moduleCompare the API key in Xenon Suite's OpenCart settings with the key configured in the OpenCart admin module.

Queue Not Processing

IssueCauseSolution
Jobs stuck in jobs tableQueue worker not runningStart a queue worker: php artisan queue:work. For production, use Supervisor or systemd.
Jobs failing repeatedlyJob logic error or timeoutCheck failed_jobs table for error messages. Run php artisan queue:retry all after fixing issues.
Queue not processing in devQUEUE_CONNECTION set to syncSet QUEUE_CONNECTION=database in .env and run a queue listener.

Permission Denied (403)

IssueCauseSolution
User cannot access a sectionMissing permission in user groupGo to Settings > User Groups > edit the group and assign the required permission.
New permissions not appearingMigration not runRun php artisan migrate to apply new permission migrations.
Admin cannot access anythingAdministrator group missing permissionsPermission migrations auto-assign to the "Administrator" group. Re-run php artisan migrate or manually assign permissions.

General Issues

IssueCauseSolution
500 Internal Server ErrorMisconfigured .env or missing APP_KEYRun php artisan key:generate. Check storage/logs/laravel.log for details.
Blank pagePHP error with display_errors offSet APP_DEBUG=true temporarily, or check PHP error log and storage/logs/laravel.log.
Storage permission errorsIncorrect directory ownershipRun chmod -R 775 storage bootstrap/cache and chown -R www-data:www-data storage bootstrap/cache.
Vite assets not loadingFrontend not builtRun npm install && npm run build. In development, run npm run dev for hot reloading.
Sessions expiring quicklySESSION_LIFETIME too lowIncrease SESSION_LIFETIME in .env (default: 120 minutes).
Images not displayingCATALOG_PUBLIC_URL misconfiguredSet CATALOG_PUBLIC_URL in .env to the public URL where OpenCart images are served.
